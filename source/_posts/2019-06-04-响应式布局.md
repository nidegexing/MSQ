# 响应式布局的实践
可以看下阮一峰大神的博客：http://www.ruanyifeng.com/blog/2012/05/responsive_web_design.html

## 为啥要做这么酷的事情
如果你跟我一样，想做一个很酷的网站，你需要适配最主流的1080P。
如果你偶尔要用笔记本访问，需要适配720P。
如果你想随时随地用手机也可以访问，那么就需要适配移动端了，移动端我们只适配1080P。
<!--more-->
## 实现思路
- bootstrap，这个很流行，但是我不会啊，哈哈哈哈
- @media，媒体查询标签。据说bootstrap也是通过这个实现的，我先写这个试试。

```
@media (max-width:1200px) {
//屏幕最大宽度小于1200，认为是移动端。
}
@media (min-width:1200px) and (max-width:1400px) {
//屏幕最小宽度大于1200，最大宽度小于1400，认为是720P。
}
@media (min-width:1400px) {
//屏幕最小宽度大于1400，认为是1080P。
}
```

### 1080P
html基础字体大小:font-size:16px;
浏览器基本高度：960px=60rem;
- 无边栏详情：宽1408px=88rem;
- 单边栏详情: 宽(1024px + 288px) = (64rem + 18rem) = 82rem;
- 双边栏详情：宽(288px + 1024px + 288px) = (18rem + 64rem + 18rem) = 100rem;
- 综合浏览: 宽(1560px = 96rem) + 自由发挥;
### 720P
- 无边栏详情: 宽1024px = 64rem;
- 单边栏详情: 宽(1024px + 288px) = (64rem + 18rem) = 82rem:
- 综合浏览: 宽(1312px = 82rem) + 自由发挥;
### 移动端
- 导航栏处理: 点击侧边栏形式弹出，[类似这个网址](https://nidegexing.github.io)
- 详情处理: 宽100%，竖排放即可。
- 快捷操作处理：固定在底部或者右下角，最好可以拖动。
